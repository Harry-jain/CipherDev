# 🎙️ Live Meeting Transcription Feature

## Overview

The Live Meeting Transcription feature enables users to record and transcribe meetings in real-time using Whisper AI models running entirely in the browser. After recording, users can generate AI-powered summaries using the already-loaded LLM and export transcriptions in Markdown or JSON format.

## ✨ Key Features

### 🔒 **100% Privacy-Preserving**
- All audio processing happens locally in the browser
- No audio data is ever sent to external servers
- Whisper models run via WebAssembly (WASM)
- Complies with CipherDev's privacy-first philosophy

### 🎯 **Voice Activity Detection (VAD)**
- Automatic speech detection using `@ricky0123/vad-web`
- Reduces unnecessary processing during silence
- Improves transcription quality with natural segment boundaries
- Real-time audio level visualization

### 🤖 **Multiple Whisper Models**
- **Whisper Large V3 Turbo** (1.5GB) - High-end devices
- **Whisper Small** (240MB) - Mid-range devices
- **Whisper Base** (140MB) - Low-end devices
- **Whisper Tiny** (75MB) - Minimal devices
- Automatic model recommendation based on device tier

### 📝 **AI-Powered Summaries**
- Generate summaries using already-loaded LLM (TinyLlama, Gemma, Llama, Phi)
- Extracts key points automatically
- Summary prioritized in exports
- Regenerate summaries with different prompts

### 📤 **Flexible Export Options**
- **Markdown** (.md) - Human-readable with summary first, full transcript at end
- **JSON** (.json) - Structured data with full metadata
- Includes timestamps, confidence scores, and metadata

## 🚀 Usage Guide

### 1. **Navigate to Transcription Page**
- Click "Transcribe" in the sidebar
- Whisper model loads automatically based on device tier

### 2. **Start Recording**
- Click "Start Recording" button
- Grant microphone permission when prompted
- Audio visualizer shows real-time levels
- Transcription appears as you speak

### 3. **Control Recording**
- **Pause**: Temporarily pause recording
- **Resume**: Continue recording
- **Stop**: End recording and finalize transcription

### 4. **Generate Summary**
- After stopping, click "Generate AI Summary"
- Uses your loaded LLM (must load from Models page first)
- Summary includes key points extracted automatically

### 5. **Export Transcription**
- Click "Export Transcription"
- Choose format:
  - **Markdown**: Summary first, full transcript at end
  - **JSON**: Structured data with metadata

### 6. **Start New Recording**
- Click "Start New Recording" to clear and begin again

## 🔧 Technical Details

### Dependencies

```json
{
  "@ricky0123/vad-web": "^0.0.7",
  "@xenova/transformers": "^2.17.2"
}
```

### Browser Requirements

- **Chrome 113+** or **Edge 113+** (for optimal performance)
- **Firefox 115+** (WASM fallback)
- **Microphone access** required
- **4GB+ RAM** recommended (8GB+ for Large models)

### Performance

| Model | Size | Device Tier | Speed | Accuracy |
|-------|------|-------------|-------|----------|
| Whisper Large V3 Turbo | 1.5GB | High | ~30 tokens/s | Excellent |
| Whisper Small | 240MB | Mid | ~20 tokens/s | Very Good |
| Whisper Base | 140MB | Low | ~15 tokens/s | Good |
| Whisper Tiny | 75MB | Minimal | ~10 tokens/s | Fair |

### Memory Usage

- **Whisper Model**: 75MB - 1.5GB (depending on model)
- **Audio Buffer**: ~10MB per minute of recording
- **Transcription Data**: ~1KB per minute of speech
- **Total**: Varies by model and recording length

## 🔐 Privacy & Security

### What Happens Locally
✅ Audio capture from microphone  
✅ Voice activity detection  
✅ Whisper transcription  
✅ LLM summary generation  
✅ Export file creation  

### What NEVER Happens
❌ Audio uploaded to servers  
❌ Transcription sent externally  
❌ Telemetry or analytics  
❌ Third-party API calls  
❌ Cloud storage  

### Data Storage
- **Whisper Models**: Cached in IndexedDB (persistent)
- **Transcriptions**: In-memory only (cleared on page refresh)
- **Exports**: Downloaded as local files only

## 🐛 Troubleshooting

### Microphone Not Working
- Check browser permissions (Settings → Privacy → Microphone)
- Ensure no other app is using the microphone
- Try refreshing the page
- Check browser console for errors

### Model Loading Fails
- Check internet connection (needed for initial download)
- Clear browser cache and reload
- Try a smaller model (Tiny or Base)
- Check available disk space

### Transcription Quality Issues
- Speak clearly and at moderate pace
- Reduce background noise
- Use a better microphone if available
- Try a larger model (Small or Large)

### Summary Generation Fails
- Ensure an LLM is loaded (visit Models page)
- Check that recording has sufficient content
- Try regenerating the summary
- Check browser console for errors

## 🚧 Known Limitations

1. **Browser Support**: WebGPU not available in all browsers (WASM fallback used)
2. **Model Size**: Large models require significant RAM and disk space
3. **Real-time Lag**: 5-10 second delay for chunk processing
4. **Language Support**: Currently optimized for English (multilingual support possible)
5. **Long Recordings**: Recordings >2 hours may cause memory issues
